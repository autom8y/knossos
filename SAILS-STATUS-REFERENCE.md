# Sails Status Display Reference

Quick reference for the sails color display feature in `ari session status`.

## Usage

```bash
# JSON output (default)
ari session status

# Text output
ari session status --output text

# YAML output
ari session status --output yaml
```

## Output Fields

### JSON/YAML
- `sails_color` (string, optional): Final sails color (WHITE, GRAY, or BLACK)
- `sails_base` (string, optional): Computed base color before modifiers

### Text Output
```
Sails: <COLOR>                    # When sails match base
Sails: <COLOR> (base: <BASE>)     # When modifiers/QA changed color
Sails: not generated              # When WHITE_SAILS.yaml missing
```

## Implementation Details

### Data Source
- Reads from `.claude/sessions/<session-id>/WHITE_SAILS.yaml`
- Generated by `ari wrap` command
- Schema version 1.0 format

### Error Handling
- Missing file: Shows "not generated", no error
- Malformed YAML: Shows "not generated", no error
- Invalid session: Existing error handling unchanged

### When Sails Differ from Base
The sails color can differ from computed base in these cases:

1. **Agent Downgrade**: Agent declares uncertainty
   - `DOWNGRADE_TO_GRAY`: White → Gray
   - `DOWNGRADE_TO_BLACK`: Any → Black

2. **Human Override**: User forces gray for manual review
   - `HUMAN_OVERRIDE_GRAY`: Any → Gray

3. **QA Upgrade**: QA session upgrades gray to white
   - Gray → White (requires adversarial tests)

### Color Meanings

| Color | Meaning | Ship? |
|-------|---------|-------|
| WHITE | High confidence - all proofs pass | ✅ Yes |
| GRAY | Unknown confidence - needs review | ⚠️  After QA |
| BLACK | Known failure - tests fail or blockers | ❌ No |
