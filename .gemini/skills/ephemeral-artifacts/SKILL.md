# Ephemeral Artifacts

> The .sos/wip/ directory is a pre-permanent scratch layer for exploration,
> design, and triage artifacts that are not yet ready for the permanent
> record.

## When to Write to .sos/wip/

Use .sos/wip/ when the artifact is exploratory, temporary, or not yet ready for its canonical location. Do not use it for platform-owned files or permanent artifacts that already have a home.

| Scenario | Type | Example Filename |
|----------|------|-----------------|
| Testing a hypothesis or time-boxed exploration | `spike` | `SPIKE-hook-cleanup.md` |
| Drafting requirements or PRD-style specs | `spec` | `SPEC-session-hardening.md` |
| Auditing code or system behavior | `audit` | `AUDIT-clew-event-coverage.md` |
| Designing architecture or context solutions | `design` | `DESIGN-ephemeral-artifacts.md` |
| Triaging issues, bugs, or priorities | `triage` | `TRIAGE-flaky-tests.md` |
| QA reports, test plans, validation results | `qa` | `QA-sync-pipeline.md` |
| Unstructured notes or temporary calculations | `scratch` | `scratch-notes.md` |

## Mandatory Frontmatter

Every new .sos/wip/ file must begin with YAML frontmatter containing a `type` field. The write guard will advise you if this is missing.

```
---
type: <spike|spec|audit|design|triage|qa|scratch>
---
```

Additional fields are permitted but not validated:

```
---
type: design
session: session-20260219-024922-e9789d71
status: draft
---
```

The type field is case-sensitive and must be one of the seven values above. No other values are accepted.

## What is NOT .sos/wip/

These files must never be written to .sos/wip/:

- **SESSION_CONTEXT.md** -- Moirai-owned, managed via `Task(moirai, "...")`. Writes are blocked.
- **SPRINT_CONTEXT.md** -- Moirai-owned, managed via `Task(moirai, "...")`. Writes are blocked.
- **events.jsonl** -- Clew-owned, written by the clew hook automatically.
- **PROVENANCE_MANIFEST.yaml** -- Pipeline-owned, generated by `ari sync`.
- **KNOSSOS_MANIFEST.yaml** -- Pipeline-owned, generated by `ari sync`.
- **ADRs** -- Permanent artifacts. Write directly to `.ledge/decisions/ADR-NNNN-*.md`.
- **Legomena** -- Permanent platform knowledge. Write to `mena/` or rite-specific `mena/`.
- **Dromena** -- Permanent commands. Write to `mena/` or rite-specific `mena/`.

If you are unsure whether something belongs in .sos/wip/ vs a canonical location, ask: "Is this ready to be permanent?" If yes, write it directly to the permanent location.

## Promotion Paths

.sos/wip/ artifacts are temporary. Once an artifact matures, promote it and delete the .sos/wip/ file.

| Type | Promotion Target | Action |
|------|-----------------|--------|
| `spike` | ADR or delete | If spike validates an approach, document the decision in an ADR. Otherwise delete. |
| `spec` | Legomenon or formalized PRD | If spec becomes platform knowledge, promote to legomenon. For product specs, promote to a formal PRD document. |
| `audit` | ADR or remediation plan | Document findings in an ADR or create a tracked remediation plan. |
| `design` | Legomenon, ADR, or implementation | Context designs become ADRs. Architectural patterns become legomena. |
| `triage` | Tracked issue or delete | Convert to a tracked issue in the project's issue tracker. |
| `qa` | Test suite or delete | Promote findings to tests or delete once addressed. |
| `scratch` | Always delete | Never promote. Scratch files are for transient calculations only. |

## File Naming Convention

Advisory (not enforced by the write guard):

- Use `PREFIX-slug.md` format where PREFIX matches the type in uppercase: `SPIKE-`, `SPEC-`, `AUDIT-`, `DESIGN-`, `TRIAGE-`, `QA-`
- Slugs should be lowercase-hyphenated descriptions of the content
- Scratch files have no required prefix convention
- Examples: `SPIKE-hook-cleanup.md`, `DESIGN-ephemeral-artifacts.md`, `TRIAGE-ci-timeout.md`

## Clew Integration

When the write guard sees a new .sos/wip/ file written with valid frontmatter, the clew hook emits an `artifact.created` event with:

- `artifact_type: ephemeral`
- `wip_type: <your type value>`
- `slug: <filename without extension>`

This makes .sos/wip/ file creation visible in the session event stream for observability and handoff tracking without requiring any extra steps from the agent.
