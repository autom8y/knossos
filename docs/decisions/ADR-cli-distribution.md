## Context

The Ariadne CLI (`ari`) needs a distribution strategy to reach end users across multiple platforms. Currently, installation requires manual compilation from source, which creates friction for adoption and makes version management difficult.

### Current State

- **Binary**: `ari` (Go CLI built from `ariadne/cmd/ari`)
- **Module**: `github.com/autom8y/knossos`
- **Target repository**: `github.com/autom8y/knossos` (post-rename)
- **Build system**: Manual `go build` commands
- **Distribution**: None formalized

### Requirements

1. **Multi-platform support**: macOS (Intel + Apple Silicon), Linux (amd64 + arm64), Windows
2. **Simple installation**: One-command install for most users
3. **Update mechanism**: Users should be able to easily upgrade to new versions
4. **Reproducible releases**: Automated, auditable release process
5. **Version tracking**: Clear versioning with changelog

### Forces

- **Developer experience**: Installation should be trivial
- **Maintenance burden**: Distribution infrastructure should be low-maintenance
- **Security**: Distribution channels should be verifiable
- **Industry alignment**: Follow patterns used by similar tools (gh, terraform, kubectl)

## Decision

### Primary Distribution Strategy

We will use **GoReleaser** for release automation with the following distribution channels:

| Channel | Platform | Priority |
|---------|----------|----------|
| **GitHub Releases** | All | Primary (canonical source) |
| **Homebrew Tap** | macOS, Linux | Primary |
| **Scoop Bucket** | Windows | Secondary (Phase 2) |
| **deb/rpm packages** | Linux | Tertiary (on demand) |

### Release Automation

**GoReleaser** will automate:
- Cross-platform binary compilation (darwin/linux/windows x amd64/arm64)
- Archive creation with checksums
- Changelog generation from git history
- GitHub Release creation with artifacts
- Homebrew formula updates
- Scoop manifest updates (Phase 2)

### Repository Structure

```
autom8y/
├── knossos/                  # Main repository (source)
│   ├── ariadne/              # CLI source code
│   ├── .goreleaser.yaml      # Release configuration
│   └── .github/workflows/
│       └── release.yml       # Release automation
└── homebrew-tap/             # Homebrew formulas
    └── Formula/ari.rb        # Generated by GoReleaser
```

### Installation Methods (Priority Order)

1. **Homebrew (macOS/Linux)** - Recommended
   ```bash
   brew tap autom8y/tap
   brew install ari
   ```

2. **GitHub Release (All platforms)** - Manual download
   ```bash
   # Download from https://github.com/autom8y/knossos/releases
   ```

3. **go install (Developers)** - From source
   ```bash
   go install github.com/autom8y/knossos/cmd/ari@latest
   ```

### Release Process

1. Create semantic version tag: `git tag -a v1.0.0 -m "Release v1.0.0"`
2. Push tag: `git push origin v1.0.0`
3. GitHub Actions triggers GoReleaser
4. GoReleaser builds, packages, and publishes to all channels

### Version Injection

Build-time version injection via ldflags:

```yaml
ldflags:
  - -X main.version={{.Version}}
  - -X main.commit={{.Commit}}
  - -X main.date={{.Date}}
```

## Consequences

### Positive

1. **Industry-standard tooling**: GoReleaser is used by gh, terraform, and other major CLIs
2. **Low maintenance**: Automated releases require only tagging
3. **Consistent experience**: Users get the same installation UX as other popular tools
4. **Cross-platform**: Single configuration handles all platforms
5. **Verifiable**: Checksums and signed releases (future enhancement)
6. **Changelog automation**: Release notes generated from commits

### Negative

1. **Additional repositories**: Requires `homebrew-tap` repository
2. **Token management**: PAT required for cross-repo publishing
3. **External dependency**: Relies on GoReleaser project maintenance
4. **Homebrew-core exclusion**: Using tap instead of official Homebrew repository

### Neutral

1. **Windows secondary**: Scoop support deferred to Phase 2
2. **No APT/YUM repos**: Linux users download manually or use Homebrew
3. **No code signing**: Initial releases unsigned (can add later)

## Alternatives Considered

### Alternative 1: Homebrew Core Submission

**Description**: Submit formula to official homebrew-core repository.

**Rejected because**:
- Long review process (weeks to months)
- Strict requirements for "notability"
- Less control over formula updates
- Requires maintaining formula outside our repos

### Alternative 2: Direct Binary Downloads Only

**Description**: Only distribute via GitHub Releases, no package managers.

**Rejected because**:
- Poor user experience (manual PATH management)
- No automatic updates
- Users must manually verify checksums
- Does not meet "simple installation" requirement

### Alternative 3: Goreleaser Pro

**Description**: Use paid GoReleaser Pro for additional features.

**Rejected because**:
- Free tier sufficient for current needs
- Pro features (monorepo support, cloud storage) not required
- Can upgrade later if needs change

### Alternative 4: Custom Release Scripts

**Description**: Build custom bash/GitHub Actions for releases.

**Rejected because**:
- Reinventing the wheel
- More maintenance burden
- Likely to have bugs GoReleaser has already solved
- Less portable across CI systems

### Alternative 5: Nix Flakes

**Description**: Distribute primarily via Nix flakes.

**Rejected because**:
- Smaller user base than Homebrew
- More complex setup for non-Nix users
- Can add as tertiary channel later if demand exists

## Implementation Notes

### Phase 1: Initial Setup

1. Create `.goreleaser.yaml` in repository root
2. Create `github.com/autom8y/homebrew-tap` repository
3. Generate PAT with `repo` scope for cross-repo publishing
4. Add `HOMEBREW_TAP_TOKEN` to repository secrets
5. Create `.github/workflows/release.yml`
6. Test with snapshot build: `goreleaser release --snapshot --clean`
7. Tag and release v1.0.0

### Phase 2: Windows Support

1. Create `github.com/autom8y/scoop-bucket` repository
2. Add Scoop configuration to `.goreleaser.yaml`
3. Add `SCOOP_BUCKET_TOKEN` to repository secrets
4. Update documentation with Windows instructions

### Phase 3: Advanced Features (On Demand)

- [ ] Code signing for macOS (notarization)
- [ ] GPG signing for artifacts
- [ ] APT/YUM repository hosting
- [ ] Nix flake support

### Configuration Checklist

- [ ] `.goreleaser.yaml` created and validated
- [ ] `homebrew-tap` repository created with README
- [ ] GitHub Actions workflow created
- [ ] PAT generated and stored as secret
- [ ] Test release with `--snapshot` flag
- [ ] Documentation updated with installation instructions
- [ ] First tagged release published
