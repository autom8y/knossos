# Mythology Concordance

> The myth is the architecture. The architecture is the myth.

This concordance maps Greek mythology to the **SOURCE implementation** of Knossos—the canonical Go code, rite definitions, and platform source that **generates** the materialized `.claude/` projections.

**Critical Distinction:**
- **SOURCE** = `/roster/` repository (versioned, canonical, what Knossos IS)
- **PROJECTION** = `.claude/` directories (gitignored, materialized by `ari sync materialize`)

---

## Core Components

### Knossos (The Labyrinth)

**Mythological Origin:**
Knossos was the palace-complex on Crete, an elaborate labyrinth built to contain the Minotaur. The labyrinth's complexity protected and trapped in equal measure.

**Knossos Implementation:**
The `/roster/` repository itself—the platform SOURCE code, not the generated `.claude/` directories.

**Key Files:**
- `/roster/` (repository root—the labyrinth incarnate)
- `/roster/internal/` (Go packages implementing platform logic)
- `/roster/rites/` (canonical rite definitions—source of practice bundles)
- `/roster/cmd/ari/` (CLI source code)

**Design Rationale:**
Knossos is the SOURCE that creates projections. The labyrinth is the palace, not the rooms within it. Per ADR-0009 Amendment (2026-01-08): The `/roster/` repository IS Knossos; `.claude/` directories are PROJECTIONS materialized into consuming projects.

---

### Ariadne (The Clew)

**Mythological Origin:**
Ariadne gave Theseus the clew (thread ball) that ensured his return from the labyrinth. The clew did not slay the Minotaur—it guaranteed safe return.

**Knossos Implementation:**
The `ari` CLI binary and its source code—the survival architecture ensuring deterministic return.

**Key Files:**
- `/roster/cmd/ari/` (CLI source, entry point: `main.go`)
- `/roster/internal/session/` (session management Go package)
- `/roster/internal/hook/clewcontract/` (clew contract types and event definitions)
- `/roster/internal/lock/` (concurrency safety mechanisms)

**Design Rationale:**
Ariadne is faithful, not intelligent. The CLI provides commands (`ari session`, `ari hook`, `ari sails`) that anchor, track, and cut the clew. Event recording (`events.jsonl`) is the physical manifestation of the thread.

---

### The Clew/Thread

**Mythological Origin:**
The physical thread that unwinds as you travel, providing a path back to the entrance.

**Knossos Implementation:**
Session state + event log (`events.jsonl`) + provenance trail.

**Key Files:**
- `/roster/internal/hook/clewcontract/` (event types, clew contract schema)
- `[session-dir]/events.jsonl` (materialized event log—PROJECTION)
- `[session-dir]/SESSION_CONTEXT.md` (session state—PROJECTION)

**Design Rationale:**
The clew is the only source of truth when context degrades. Events record decisions, tool calls, artifacts, errors. The Ship of Theseus problem: identity persists through transformation because the clew remembers what Theseus forgets.

---

### Theseus (The Navigator)

**Mythological Origin:**
Theseus entered the labyrinth to slay the Minotaur, using Ariadne's thread to ensure return.

**Knossos Implementation:**
The main Claude Code thread—the agentic intelligence that makes decisions, summons heroes, creates artifacts.

**Key Files:**
N/A (Theseus is the LLM agent itself, not a file)

**Design Rationale:**
Theseus has agency but amnesia. The clew compensates. Heroes (specialist agents) are summoned mid-journey via Task tool delegation, arriving with rich context from the clew.

---

### Heroes (Specialist Agents)

**Mythological Origin:**
Greek heroes summoned for specific labors (Heracles for strength, Odysseus for cunning).

**Knossos Implementation:**
Specialist agents invoked via Task tool, defined in rite manifests.

**Key Files:**
- `/roster/user-agents/` (canonical agent source definitions)
- `/roster/rites/[rite-name]/agents/` (rite-specific agent definitions)
- `.claude/agents/` (materialized projections—generated by `ari sync materialize`)

**Design Rationale:**
Heroes don't walk the whole labyrinth; they're summoned mid-journey. Better clew → better summoning context → better hero performance. Agents defined in SOURCE, materialized to PROJECTION.

---

### The Moirai (The Fates)

**Mythological Origin:**
Clotho spins the thread of life, Lachesis measures its allotment, Atropos cuts it when complete.

**Knossos Implementation:**
Centralized session lifecycle agent with event-driven activation.

**Key Files:**
- `/roster/user-agents/moirai.md` (agent definition—SOURCE)
- `/roster/internal/session/` (Go implementation of lifecycle state machine)
- `.claude/agents/moirai.md` (materialized projection)

**Design Rationale:**
Only the Moirai may mutate session state. Clotho activates on `session_start`, Lachesis on state mutations, Atropos on `session_end`. Write guards block direct edits to `*_CONTEXT.md` files (ADR-0005, ADR-0013).

---

### The Inscription (CLAUDE.md)

**Mythological Origin:**
Words carved in stone at the labyrinth's entrance, telling travelers what to expect.

**Knossos Implementation:**
The `CLAUDE.md` file that declares available rites, agents, execution mode, and hooks.

**Key Files:**
- `/roster/knossos/templates/CLAUDE.md.tpl` (canonical template—SOURCE)
- `/roster/internal/inscription/` (Go package for Inscription generation)
- `.claude/CLAUDE.md` (materialized projection per project)

**Design Rationale:**
The labyrinth speaks at entry. SessionStart hooks read the Inscription to prepare Theseus. Knossos-managed sections (delimited by `<!-- KNOSSOS:START -->` ... `<!-- KNOSSOS:END -->`) are regenerated by `ari sync inscription`. Custom sections outside delimiters are preserved.

---

### White Sails (Confidence Signal)

**Mythological Origin:**
Aegeus watched from the cliff for his son's return. White sails = success, black sails = death. Theseus forgot to change the sails; Aegeus, seeing black, threw himself into the sea.

**Knossos Implementation:**
Computed confidence signal (WHITE/GRAY/BLACK) generated at session wrap.

**Key Files:**
- `/roster/internal/sails/` (Go package implementing confidence computation)
- `ari sails check` (CLI command)

**Design Rationale:**
Solves the Aegeus problem (false confidence). Three-color system prevents gaming. GRAY is honesty about uncertainty. Anti-gaming rules: no self-upgrade, proofs required for WHITE, QA review can elevate GRAY → WHITE.

---

### Rites (Practice Bundles)

**Mythological Origin:**
Ceremonies and invocations—structured practices for specific purposes.

**Knossos Implementation:**
Manifest-driven bundles of agents, skills, hooks, and workflows.

**Key Files:**
- `/roster/rites/` (canonical source definitions)
- `/roster/rites/[rite-name]/manifest.yaml` (rite composition definition)
- `/roster/internal/rite/` (Go package for rite loading and validation)
- `.claude/rites/` (materialized projections)

**Design Rationale:**
Rites are flexible compositions. Simple rites (skills-only) are valid. Invoke rites mid-journey (cheap) or swap rites (expensive full context switch). Rite source lives in `/roster/rites/`, projections materialize to `.claude/rites/` via `ari sync materialize`.

---

### Daedalus (The Builder)

**Mythological Origin:**
Daedalus built the labyrinth and crafted wings to escape it. Master architect and inventor.

**Knossos Implementation:**
The `forge-rite`—practice for building agents, tools, and platform infrastructure.

**Key Files:**
- `/roster/rites/forge/` (forge-rite source)
- `/roster/rites/forge/manifest.yaml`

**Design Rationale:**
Designed complexity is Daedalus's gift. Not all complexity is debt; some is intentional architecture. The forge-rite enables platform self-improvement and tool creation.

---

### Pythia (The Oracle)

**Mythological Origin:**
The Oracle at Delphi consulted before great journeys. Unlike the historical cryptic Pythia, Knossos's oracle speaks clearly.

**Knossos Implementation:**
Orchestrator agents that provide work breakdown, specialist routing, and checkpoint guidance.

**Key Files:**
- `/roster/user-agents/orchestrator.md` (base orchestrator pattern)
- Rite-specific orchestrators (e.g., `/roster/rites/docs/agents/orchestrator.md`)

**Design Rationale:**
Consulted via `/consult` for workflow routing. Orchestrators decompose initiatives into phases and route to specialist heroes. Clear guidance, not cryptic prophecy.

---

### Athens (Home / Main Branch)

**Mythological Origin:**
Theseus's home—where he returns after the journey.

**Knossos Implementation:**
The `main` branch—the destination of merged PRs.

**Key Files:**
N/A (Git branch, not a file)

**Design Rationale:**
Return = merged PR. The ship is the PR; the merge is the homecoming. Until work reaches `main`, the journey is incomplete.

---

### Naxos (Shore of Abandonment)

**Mythological Origin:**
The island where Theseus abandoned Ariadne after she saved him.

**Knossos Implementation:**
Orphaned sessions—created but never wrapped, abandoned by their creator.

**Key Files:**
- `/roster/internal/naxos/` (Go package for orphan detection)
- `ari naxos` commands (orphan cleanup)

**Design Rationale:**
Sessions left too long accumulate as technical debt. Naxos detection identifies stale sessions for cleanup or resumption. Represents unfinished business and broken promises.

---

### Dionysus (The Transformer)

**Mythological Origin:**
Dionysus found Ariadne on Naxos and made her divine—transformation through elevation.

**Knossos Implementation:**
Code review process that elevates work from isolation to merged canon.

**Key Files:**
- Review workflows (rite-specific)
- QA upgrade mechanism in White Sails (independent review elevates GRAY → WHITE)

**Design Rationale:**
Dionysus transforms abandonment into elevation. Code review validates, improves, and blesses work for union with `main`. Independent validation prevents self-deception.

---

### Minos (The Commissioner)

**Mythological Origin:**
King Minos commissioned the labyrinth and demanded tribute from Athens.

**Knossos Implementation:**
Stakeholders who create initiatives and demand status reports.

**Key Files:**
- `/roster/internal/tribute/` (status report generation)

**Design Rationale:**
Tribute = demos and status reports. Periodic demonstration that progress is made, the Minotaur will eventually fall, return is approaching.

---

### The Minotaur (The Task)

**Mythological Origin:**
The beast at the labyrinth's heart—the reason for the journey.

**Knossos Implementation:**
The initiative, feature, or task being pursued.

**Key Files:**
Defined in `SESSION_CONTEXT.md` as the initiative

**Design Rationale:**
The Minotaur is not evil—it's simply the challenge. Size varies (bug fix vs platform migration). The clew cares not for the Minotaur's size, only that Theseus returns.

## Materialization Flow

**The Fundamental Truth:**

```
SOURCE (versioned, canonical)           PROJECTION (gitignored, generated)
/roster/rites/                       →  .claude/rites/
/roster/user-agents/                 →  .claude/agents/
/roster/user-skills/                 →  .claude/skills/
/roster/internal/hook/ (Go) + /roster/rites/[rite]/hooks/ (scripts) → .claude/hooks/
/roster/knossos/templates/           →  .claude/CLAUDE.md (rendered)
/roster/rites/[rite]/manifest.yaml   →  .claude/ACTIVE_RITE
```

**Command:** `ari sync materialize` reads SOURCE and writes PROJECTION.

Knossos (the SOURCE) generates `.claude/` (the PROJECTION). The labyrinth creates the rooms; the rooms are not the labyrinth.

---

## Mythology as Architecture

The mythology is not decoration—it's the design language. Each mythological element encodes:

1. **Architectural intent**: Why components exist
2. **Relationship patterns**: How components interact
3. **Constraint awareness**: What limits shape design
4. **Failure modes**: What happens when principles are violated

When you understand the myth, you understand the system. When you violate the myth, you violate the architecture.
